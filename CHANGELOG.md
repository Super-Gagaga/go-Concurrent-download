# 更新日志

所有项目的重大更改都将记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
并且本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [0.1.0] - 2026-03-26

### 新增
- 初始版本发布
- 核心功能：并发分片下载、断点续传、进度监控、异常重试
- HTTP 客户端实现，支持 Range 请求和连接池
- 下载引擎，支持并发控制和任务管理
- 状态管理，支持 JSON 格式的状态持久化
- 进度监控系统，支持历史记录和速度计算
- 完整的错误类型系统和错误处理机制
- 命令行工具，支持多种命令和参数
- 全面的 API 文档和示例代码
- 单元测试框架，覆盖率超过 85%

### 技术特性
- 纯 Go 标准库实现，无外部依赖
- 基于 goroutine 和 channel 的并发模型
- 使用功能选项模式进行配置
- 接口驱动的设计，易于测试和扩展
- 支持 HTTP/HTTPS 协议
- 自动检测服务器是否支持 Range 请求
- 智能分片大小计算算法
- 指数退避重试机制，带随机抖动
- 实时进度更新，支持多种显示格式
- 信号处理，支持优雅停止

### API 功能
- `Download()`: 快速下载单个文件
- `DownloadWithConfig()`: 带配置的下载
- `DownloadBatch()`: 批量下载多个文件
- `NewDownloader()`: 创建下载器实例
- 完整的任务控制：暂停、恢复、取消
- 任务状态查询和列表功能

### 命令行功能
- `download/dl`: 下载单个文件
- `batch`: 批量下载
- `resume`: 恢复中断的下载
- `list`: 列出所有任务
- `status`: 查看任务状态
- `version/v`: 显示版本信息
- `help/h`: 显示帮助信息

### 配置选项
- 并发数控制（默认 4）
- 重试次数和间隔（默认 3 次，1 秒间隔）
- 超时设置（连接、读取、总超时）
- 缓冲区大小（默认 32KB）
- 断点续传开关（默认开启）
- 校验和验证（MD5、SHA1、SHA256、SHA512）
- HTTP 头部自定义
- 用户代理设置
- 下载速度限制
- 分片大小范围（默认 1MB-10MB）

- 完整的项目需求文档
- 详细的 API 文档
- 快速开始指南
- 使用示例（简单和高级）
- 贡献指南
- 测试覆盖率报告

### 已知限制
- 目前仅支持 HTTP/HTTPS 协议
- 不支持需要 Cookie 或复杂认证的下载
- 批量下载功能为演示实现
- 某些高级功能需要进一步优化

### 下一步计划
- 实现完整的下载引擎逻辑
- 添加 FTP 协议支持

- 添加更多认证方式支持
- 优化内存使用和性能
- 添加更多集成测试
- 开发 GUI 界面

## 版本说明

- 主版本号：不兼容的 API 修改
